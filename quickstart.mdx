You can use ACGTI in two ways: visit the live site and take the quiz immediately, or clone the repository and run it locally to develop, test, or contribute. Both paths are covered below.

## Take the quiz online

<Steps>
  <Step title="Open the site">
    Go to [acgti.tianxingleo.top](https://acgti.tianxingleo.top/). No account or sign-up required.
  </Step>
  <Step title="Read the introduction and start">
    The intro page explains the quiz format — 39 scenario questions on a seven-point scale. When you are ready, press the start button to begin.
  </Step>
  <Step title="Answer all 39 questions">
    Each question presents a scenario. Drag the slider or tap a point on the scale from −3 to +3 to record how much you agree or disagree with the scenario. You can go back and change any answer before submitting. A progress bar at the top shows how far along you are.
  </Step>
  <Step title="Get your result">
    After answering all questions, your result page shows your character code, the matched archetype, four dimension sliders (Energy, Mind, Nature, Tactics), and a short character profile. Up to three runner-up characters are also shown.
  </Step>
  <Step title="Export and share">
    Tap the share/export button to download your result as a PNG poster. Send it to friends or post it anywhere — the image contains your character code, archetype, and dimension bars.
  </Step>
</Steps>

## Run ACGTI locally

ACGTI is a pure-frontend app built with **Vue 3**, **TypeScript**, **Vite**, and **Tailwind CSS**. There is no backend to configure.

### Prerequisites

- Node.js 18 or later
- npm (comes with Node.js)

### Install and start the dev server

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/tianxingleo/ACGTI.git
    cd ACGTI
    ```
  </Step>
  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>
  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    Vite will print a local URL (typically `http://localhost:5173`). Open it in your browser to see the app with hot-module replacement enabled.
  </Step>
</Steps>

### Build for production

```bash
npm run build
```

The compiled output is written to `dist/`. The Vite config sets `base: './'`, so all asset paths are relative — the `dist/` folder can be dropped into any static host without further configuration.

### Deploy to Cloudflare Pages

ACGTI's official deployment runs on Cloudflare Pages. To deploy your own fork:

<Steps>
  <Step title="Push to GitHub">
    Make sure your fork is pushed to a GitHub repository.
  </Step>
  <Step title="Connect to Cloudflare Pages">
    In the Cloudflare dashboard, create a new Pages project and connect it to your GitHub repository.
  </Step>
  <Step title="Configure build settings">
    Set the build command to `npm run build` and the output directory to `dist`. Cloudflare Pages will build and deploy automatically on every push to `main`.
  </Step>
</Steps>

<Tip>
  The repository also includes a GitHub Actions workflow that runs `npm ci` and `npm run build` on every push to `main` or `dev` and on all pull requests. This catches build errors before they reach production. Tags matching `v*` trigger an automated GitHub Release that packages `dist/` as a zip file.
</Tip>

## Tech stack summary

| Layer | Technology |
| --- | --- |
| UI framework | Vue 3 (Composition API) |
| Language | TypeScript |
| Build tool | Vite |
| Styling | Tailwind CSS |
| Routing | Vue Router (hash history) |
| Hosting | Cloudflare Pages |
| State | Vue composables (`useQuiz.ts`) |
| Persistence | `localStorage` (last result only) |

<Note>
  All quiz data — questions, archetypes, and characters — is shipped as static JSON files under `src/data/`. The scoring engine (`src/utils/quizEngine.ts`) loads them lazily on first use via dynamic imports, keeping the initial bundle small.
</Note>
